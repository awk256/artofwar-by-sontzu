
# 概要

孫子の「五事」を、現代のプロジェクト・レビューや投資判断に使える**Go/No-Go判定リスト**として再構成しました。客観的なスコアリングを通じて、戦略の「穴」を特定します。

# チェック項目

### 1. 道（Why | 理念・一体感）

* [ ] なぜ「今これをやるのか」をメンバー全員が自分の言葉で説明できるか
* [ ] 経営・現場・顧客の3者の利益が、同じ方向を向いているか
* [ ] 現場が「無理やりやらされている」という受動的な状態になっていないか

> **リスク：** ここにチェックが入らない場合、**困難に直面した際にチームが内部崩壊**します。

### 2. 天（When | タイミング）

* [ ] 市場は「早すぎる（教育コスト過多）」または「遅すぎる（コモディティ化）」状態ではないか
* [ ] 利用する技術の成熟度や、関連する法規制の整備状況は味方しているか
* [ ] 成功の条件が「外部要因（景気や競合のミス）」に依存しすぎていないか

> **リスク：** ここにチェックが入らない場合、**「正しい戦略だが、時期が悪くて死ぬ」** という最悪の結果を招きます。

### 3. 地（Where | 戦場・構造）

* [ ] 競合の得意領域や、圧倒的な先行者がいる土俵で正面衝突していないか
* [ ] 自社の保有データや既存資産が「構造的な優位性」として機能しているか
* [ ] その戦場（市場・プラットフォーム）での「勝ち筋」を数式やロジックで説明できるか

> **リスク：** ここにチェックが入らない場合、**どれほど努力しても報われない消耗戦**になります。

### 4. 将（Who | リーダー）

* [ ] 意思決定が迅速であり、朝令暮改や責任転嫁が起きていないか
* [ ] 失敗時の責任の所在が明確であり、リーダーにそれを引き受ける覚悟があるか
* [ ] 専門外の領域についても、現場の意見を「正しく聴く」謙虚さと知性があるか

> **リスク：** ここにチェックが入らない場合、**優秀なメンバーから先にチームを離脱**します。

### 5. 法（How | 制度・実行力）

* [ ] 戦略を具体的なタスクに分解し、現場が迷いなく実行できる仕組みがあるか
* [ ] 評価制度やリソース配分が、掲げている方針と矛盾していないか
* [ ] 現場のフィードバックが迅速に仕組みへ反映される「学習ループ」があるか

> **リスク：** ここにチェックが入らない場合、**戦略は美しいスライドの中だけで終わり**ます。

# 実践：簡易スコアリングシート

各項目を **0〜2点** で評価し、プロジェクトの「健全性」を可視化します。

| 項目 | 0点：極めて不安 | 1点：課題あり | 2点：万全である |

| **天 (When)** |  |  |  |
| **地 (Where)** |  |  |  |
| **将 (Who)** |  |  |  |
| **法 (How)** |  |  |  |
| **合計点** |  |  | **/ 10点** |

### 判定基準

* **8–10点：実行可（Go）**
* 自信を持って推進してください。


* **5–7点：修正必須（Conditional Go）**
* 0点または1点の項目が「致命的なボトルネック」になります。そこを解消するまでアクセルを踏んではいけません。


* **0–4点：着手すべきでない（Stop）**
* 撤退、あるいは戦略の根本的な再定義が必要です。


### アーキテクトへのアドバイス

> 五事のバランスが崩れている場合、技術（How）だけで解決しようとするのは下策です。特に「道（Why）」や「将（Who）」に欠陥がある場合、どんなに優れたAIを導入してもプロジェクトは失敗します。
