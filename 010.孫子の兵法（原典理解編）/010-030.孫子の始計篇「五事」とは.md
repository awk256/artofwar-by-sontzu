# 概要

孫子が冒頭の「始計篇」で説くのは、戦う前に必ず比較・評価すべき**5つの基本的評価軸（The Five Factors）** です。これらは、いかなるプロジェクトにおいても成否を分ける **「勝算の構成要素」** となります。


# 1. 道（どう | Why）：理念・目的への一致

**原典の意味：** 上（リーダー）と下（民・兵）が同じ目的を共有し、死生を共にできるほどの一体感があること。

* **現代的解釈：** プロジェクトのビジョンや「なぜAIを使うのか」という大義への納得感。
* **AI設計への示唆：** 開発者やユーザーがそのAIの存在意義を信じられるか。

> **指標：** 「チームが迷わず、理由なく離脱しない状態にあるか」

# 2. 天（てん | When）：時・タイミング

**原典の意味：** 季節、天候、昼夜といった、自らの意志では変えられない外部の自然条件。

* **現代的解釈：** 市場のトレンド、技術の成熟度（GAIの進化速度）、法規制の動向。
* **AI設計への示唆：** 精度やコストの面で、その技術は「今」実装すべきレベルに達しているか。

> **指標：** 「時流は味方しているか？ 今やるべき合理的理由があるか？」

# 3. 地（ち | Where）：場所・構造

**原典の意味：** 距離、地形の険しさ、広さといった地理的条件。

* **現代的解釈：** 業界内のポジション、既存システムとの距離感（データ基盤の整備状況）、戦う市場の構造。
* **AI設計への示唆：** 自社が持つデータセットや既存資産が、競合に対して優位に働く場所を選んでいるか。

> **指標：** 「自社にとって有利なフィールド（プラットフォーム）で戦っているか？」

# 4. 将（しょう | Who）：リーダーの資質

**原典の意味：** 指揮官の智（知略）、信（信頼）、仁（思いやり）、勇（決断力）、厳（規律）の5つ。

* **現代的解釈：** 意思決定者のリーダーシップ、AIの特性（不確実性）を理解した上での説明責任。
* **AI設計への示唆：** トラブル発生時に責任を取り、撤退や継続を冷徹に判断できるリーダーがいるか。

> **指標：** 「このリーダーに、プロジェクトの命運を預けられるか？」

# 5. 法（ほう | How）：制度・運営

**原典の意味：** 組織編成、軍制、補給路の確保、予算管理といった運用の仕組み。

* **現代的解釈：** 開発プロセス、ガバナンス、権限設計、評価制度。
* **AI設計への示唆：** AIが生成したアウトプットを検証し、改善し続ける「持続可能な運用フロー」が定義されているか。

> **指標：** 「属人化せず、ルールに基づいて実行・改善できる体制があるか？」


# 五事による「勝算のシミュレーション」

孫子は、この「五事」を比較することで、戦う前に勝敗を知ることができると断言しています。

| 要素           | 問い       | 失敗するプロジェクトの特徴          |
| ------------ | -------- | ---------------------- |
| **道(Why)**   | なぜやるのか？  | 上層部の思いつきで、現場が冷めている     |
| **天(When)**  | 今なのか？    | 技術が未熟、あるいは規制でがんじがらめ    |
| **地(Where)** | どこでやるのか？ | 勝ち目のないレッドオーシャン、データ不毛地帯 |
| **将(Who)**   | 誰がやるのか？  | リスクを理解せず、他責にするリーダー     |
| **法(How)**   | どうやるのか？  | 運用ルールがなく、行き当たりばったりの実装  |
