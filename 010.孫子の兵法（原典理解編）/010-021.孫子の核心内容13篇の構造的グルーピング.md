
# 1.計画・戦略：戦う前に勝負を決める

* **始計篇 / 作戦篇 / 謀攻篇**
* **本質：** 勝敗は現場ではなく「準備」と「見積もり」で決まる。
* **現代の視点：** 投資対効果（ROI）の算定、市場参入の可否判断。

#  2.組織・オペレーション：戦いの「型」を作る

* **軍形篇 / 兵勢篇 / 虚実篇 / 軍争篇 / 九変篇**
* **本質：** 固定的な戦略に固執せず、構造と変化を力に変える。
* **現代の視点：** 開発プロセス、スケーラビリティ、アジャイルな方向転換。

# 3.情報・環境理解：戦場の「理」を知る

* **行軍篇 / 地形篇 / 九地篇**
* **本質：** 環境を誤認すれば、どれほど強力な組織でも必ず敗れる。
* **現代の視点：** プラットフォーム選定、業界規制の把握、フェーズ別マネジメント。

# 4.特殊手段・諜報：情報を制す

* **火攻篇 / 用間篇**
* **本質：** 破壊のリスクを制御し、事実に基づいたインサイトを得る。
* **現代の視点：** リスク管理、データ分析、ユーザーリサーチ。

# カテゴリ別：現代ビジネス・IT対応表

ソリューションアーキテクトやシニアSEが、実務上の課題に直面した際の逆引きリストです。

| カテゴリ      | 該当篇   | 現代的解釈・IT実務への適用           |
| --------- | ----- | ------------------------ |
| **戦略設計**  | 始計・謀攻 | サービスコンセプト、差別化、Go/No-Go判断 |
| **経営資源**  | 作戦    | 予算管理、開発期間の設定、MVPの定義      |
| **組織設計**  | 軍形・兵勢 | 評価制度、チーム文化、マイクロサービス化     |
| **競争戦略**  | 虚実・軍争 | ニッチ戦略、ポジショニング、先行/後発利得    |
| **適応戦略**  | 九変    | ピボット、技術スタックの刷新、仕様変更対応    |
| **現場判断**  | 行軍・地形 | クラウド選定、インフラ監視、ドメイン分析     |
| **状況別統治** | 九地    | 立ち上げ期/安定期/撤退期のマネジメント変更   |
| **情報戦**   | 用間    | データレイク構築、BI、ログ活用、競合調査    |
| **リスク管理** | 火攻    | セキュリティ事故対応、炎上対策、負債の清算    |

>ニッチ戦略とは、大手企業が参入しないような「隙間（ニッチ）市場」を狙い、特化した製品・サービスを提供することで、限定された領域で高いシェアと利益を確保する経営手法である。

# 現代的要約

本書が提唱する「孫子ベースのAI戦略」の根幹はここにあります。

> **「リソース制約下において、情報と適応力を駆使し、最小コストで最大成果を得るための意思決定アルゴリズム」**

これは、不確実性が高く、リソースの奪い合いが激しいAI開発の現場において、そのまま　**「勝利の設計図」**　となります。
