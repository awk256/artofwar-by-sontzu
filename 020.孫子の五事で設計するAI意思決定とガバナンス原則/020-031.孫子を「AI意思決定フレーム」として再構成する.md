
### 結論から

**孫子の兵法は、  
AIに判断を委ねる前に「人が決めるべきこと」を体系化した書である。**

AI時代に再評価される理由は、  
孫子が **実行の方法ではなく、判断の条件**を扱っている点にある。

# 孫子をAI意思決定の3層フレームで読む

### ① 判断前層（人が決める）

**＝ 孫子の中核**

- 目的は何か（勝つとは何か）

- 戦う／使う価値はあるか

- 失敗しても耐えられるか


👉 **始計・五事・謀攻**

### ② 判断実行層（AIが処理する）

- 条件内での最適化

- 予測・分類・推薦


👉 **兵勢・虚実・軍争**


### ③ 例外・停止層（人が介入する）

- 想定外の兆候検知

- 状況変化への切替

- 強制停止・撤退


👉 **九変・行軍・火攻**


# 孫子 × AI意思決定 対応表

| 孫子の概念 | AI意思決定での意味 |
| ----- | ---------- |
| 始計    | 問題定義・制約設定  |
| 五事    | AI利用の前提条件  |
| 謀攻    | 使わない判断     |
| 虚実    | 探索空間の制限    |
| 九変    | 例外処理・切替    |
| 用間    | データと情報設計   |
| 火攻    | 高リスク操作の禁止  |

## なぜこのフレームがAIに効くのか

AIは

- **どう最適化するか**は得意

- **何を最適化してよいか**は決められない

孫子はまさに、  
**「最適化してはいけない領域」を先に定義する思想**。

👉 これはそのまま  
**AIガバナンス・Human-in-the-loopの原型**になる。

- [[020-032.Human-in-the-loop（HITL）]]

## 孫子的AI意思決定の特徴

- 全自動を目指さない

- 勝率より**損失回避**を優先

- 撤退判断を成功とみなす


## 一文要約

> **孫子の兵法とは、  
> AIに考えさせる前に  
> 人が考え終えておくための意思決定フレームである。**
