
# 概要

孫子の思想をAI設計にインストールすることで、組織やエンジニアの視点は「点（技術実装）」から「面（戦略・統治）」へと拡張されます。この変化は、単なるスキルの習得ではなく、**意思決定のOSをアップデートする**ことに他なりません。

# 5つのマインドセット・シフト

### 1. 技術中心から「目的志向」へ（道｜Why）

* **変化：** 「最新のAIを使いたい」という手段への執着が消え、**「その自動化は、我々のビジョンや価値に直結しているか？」**　という問いが先行します。
* **結果：** 流行に流されない、真に費用対効果の高いAI活用が選択されます。

### 2. 性能評価から「機微の見極め」へ（天｜When）

* **変化：** 精度ベンチマークの数値だけでなく、**「今、市場や法規制、ユーザーの成熟度はAIを受け入れる準備ができているか？」**　というタイミングを重視します。
* **結果：** 時期尚早な導入による事故や、陳腐化した技術への執着を防ぎます。

### 3. データ至上主義から「現場適応」へ（地｜Where）

* **変化：** 「モデルさえ良ければ勝てる」という幻想を捨て、**「そのAIは、この業務現場の制約や文化（地形）の中で機能するか？」**　を考え抜きます。
* **結果：** 開発と現場の乖離が埋まり、実効性の高いラストワンマイルの設計が可能になります。

>ラストワンマイル（Last One Mile）とは、物流の最終拠点（配送センター等）から、消費者や企業といったエンドユーザーの手元に商品が届くまでの「最後の1区間」を指す言葉。

### 4. 自動化の妄信から「責任ある介入」へ（将｜Who）

* **変化：** 「AIがやったことだから」という逃げを排し、**「人間がどの粒度までなら判断の責任を引き受けられるか」**　という境界線を設計します。
* **結果：** 責任の所在が明確になり、経営層が安心してアクセルを踏めるガバナンスが構築されます。

### 5. 成功の追求から「失敗の制御」へ（法｜How）

* **変化：** 100%の精度を目指す不毛な戦いをやめ、**「80%の精度でも、失敗時に被害を最小化し、即座に停止・修復できる仕組み」**　を優先します。
* **結果：** 事故に強い、回復力（レジリエンス）の高いシステムが標準となります。

# まとめ：不敗のAI設計への転換

孫子をインストールした後の思考は、一言で言えば　**「賢いAIを作ろう」から「負けないAIを設計しよう」**　へと進化します。

### 思考のBefore / After

| 項目         | Before（技術・戦術中心） | After（孫子・戦略中心）         |
| ---------- | --------------- | ---------------------- |
| **関心事**    | 精度、パラメータ数、SOTA  | **適合性、ガバナンス、ROI**      |
| **目標**     | **全自動化**の達成     | **制御可能**な自動化           |
| **失敗への態度** | 精度向上でカバーする      | **失敗条件**を定義し、ガードレールを張る |
| **撤退判断**   | 予算が切れるまで継続      | 予兆を検知し、**早期にピボット・撤退**  |

# 一文まとめ

孫子は「AI活用の良し悪し」を論じるための知識ではありません。

1. **問題設定を疑う：** 「解けるか？」より「解くべきか？」
2. **制御を先に考える：** 精度よりガードレール。
3. **撤退を前提にする：** 成功条件より先に、失敗時の「停止プロトコル」を決める。

この思考の転換こそが、AIという強力な「火」を、自らを焼き尽くすことなく使いこなすための唯一の道です。
