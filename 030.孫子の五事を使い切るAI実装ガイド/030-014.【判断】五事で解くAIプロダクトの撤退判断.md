
# 概要

孫子の兵法において、最も困難で、かつ将の真価が問われるのは「退き時」の判断です。AI開発における撤退は、プロジェクトの失敗を意味しません。むしろ、**「五事（前提条件）」の崩壊をいち早く察知し、貴重な軍財（予算・人材・時間）を壊滅前に救い出す「知的な防衛成功」**　と再定義すべきです。

# 五事で自問する「撤退のレッドフラグ」

以下の5つの問いのうち、**2つ以上が「NO」**　である場合は、プロジェクトの強行ではなく「勇気ある転進（リソースの再配置）」を検討すべきシグナルです。

### **道（Why）：目的の健全性**
- [ ] 当初の目的を、今もチーム全員が信じているか？
- [ ] AI導入そのものが目的化し、現場の課題解決が置き去りになっていないか。

### **将（Who）：責任の所在**
- [ ] トラブルの責任を、AIや外部のせいにし始めていないか？
- [ ] 責任者が判断を回避し、システムの挙動を誰も制御しようとしない「無責任状態」に陥っていないか。

### **天（When）：時機の妥当性**
- [ ] 今のコストと技術で、本当に黒字化（あるいは投資回収）の出口が見えているか？
- [ ] 法規制や新技術の登場により、完成時には技術的に陳腐化、あるいは赤字垂れ流しになる懸念はないか。

### **地（Where）：環境の適合性**
- [ ] 現場のデータと業務フローを、AIに合わせて無理に歪めていないか？
- [ ] 土壌（データ品質）が悪すぎて、改良を重ねても「泥沼」から抜け出せない確信があるか。

### **法（How）：統治の実現性**
* [ ] 異常時に、迷わず停止・切替ができる準備（キルスイッチ）が整っているか？
* [ ] ブラックボックス化が進み、もはや人間が「統治（ガバナンス）」できない領域に達していないか。

# リーダーへの提言

孫子の説く「不敗」とは、勝てない状況で戦わない知性を指します。

* **前提の崩壊を認める：** 「NO」が重なるのは技術の敗北ではなく、戦うための「前提条件」が崩れたことを意味します。
* **漂流を止める：** 前提が崩れたまま進むのは、目的のある「進軍」ではなく、ただの「漂流」です。
* **リソースを救う：** ここで踏みとどまり、浮いたリソースを別の「勝算ある地」へ再配置することこそが、将（責任者）としての最大の功績になります。

> **「まだいける」という執着を捨て、「五事」という冷徹な物差しでプロジェクトを検分してください。**

# 撤退を「次なる勝利の準備」に変える思考

孫子の「勝てぬと知りながら戦うのは、将の罪」という言葉は、不毛な戦いを続けることこそが最大の損失であることを説いています。

### 1. **「損切り」を「軍の温存」と読み替える**
「これだけ投資した（サンクコスト）」は、撤退を阻む最大のノイズです。今止めることで守れる「残りの予算とメンバーの士気」を、**「次の勝てる戦場（プロジェクト）」への軍資金**として再評価してください。

### 2. **「敗北の記録」を「最強の情報（間）」に変える**
撤退時に「なぜ五事が崩れたのか」を冷徹に分析し、組織の資産として残します。この「失敗の知見」こそが、競合が持たない独自のインテリジェンスとなり、次回の勝率を飛躍的に高めます。

### 3. **「不戦の勝利」を称賛する**
AIを使わずとも、プロセスの見直しだけで解決できることが判明して撤退するなら、それは「最小のコストで目的を達した最高の勝利」です。AIを使わなかったことを誇りとして記録しましょう。
