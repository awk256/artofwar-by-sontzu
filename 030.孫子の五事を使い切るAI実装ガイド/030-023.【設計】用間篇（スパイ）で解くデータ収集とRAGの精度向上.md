
# 概要

孫子は「敵情を知らざる者は、仁の至りなり（情報を惜しんで敵情を知ろうとしないのは、指導者失格である）」と説きました。AI、特にRAGにおいては、**「どのデータ（間諜）を信じるか」** がAIの出力（判断）のすべてを決定します。

本項では、情報の信憑性をシステム的に評価し、精度の高い回答を導き出すための「用間（インテリジェンス）」の設計思想を扱います。

# 1. 五つの間（五間）をRAGのデータソースに例える

孫子は5種類のスパイを使い分けることを説きました。これをAIの参照ソースに置き換えると、情報の「性質」と「リスク」が見えてきます。

* **郷間（地域の住民）：** 社内の一般ドキュメントやWiki。量は多いが、情報の鮮度や正確性にバラつきがある。
* **内間（敵の官吏）：** 専門部署のDBや基幹システム。正確だが、アクセス権限や構造の理解が必要。
* **反間（敵のスパイを寝返らせる）：** 外部のWeb情報や競合の公開データ。利用には高度な検証（フィルタリング）が必須。
* **死間（偽情報を流す）：** 誤情報や古いデータ。これにAIが触れると、誤答（ハルシネーション）の原因となる。
* **生間（帰還して報告する）：** リアルタイムのAPIや現場からのフィードバック。最新の状況を伝える最も重要な情報。


# 2. 「反間」を制する者がRAGを制す

孫子は、スパイ戦の核心は「反間（二重スパイ）」を使いこなすことにあると説きました。これをRAGに適用すると、**「データのメタデータ管理とクリーニング」** に相当します。

* **情報の出所（Source）の格付け：** すべてのデータを等しく信じるのではなく、情報の信頼度（信頼スコア）を付与し、高スコアの情報を優先してAIに参照させます。
* **検証（Cross-Check）：** 異なるソースから得られた情報を照合し、矛盾がある場合は「確実な情報が見つからない」と回答させる制御を組み込みます。

# 3. 「用間」のコストを惜しまない

「先知（あらかじめ知ること）」には莫大なコストがかかります。RAGにおいて、モデルの性能向上（What）に予算を割く前に、**「データの整備（用間）」** にリソースを集中すべきです。

* **データの前処理（ETL）：** 汚れたデータ（死間）を排除するためのクレンジングに投資する。
* **ナレッジの構造化：** AIが理解しやすい形に情報を整理することは、孫子が戦う前に戦場を下調べするのと同義です。

# 4. 信頼性をシステムに組み込む3つの手法

1. **引用元（Citation）の義務化：** 出力を「生間（生の情報）」として扱うため、回答の根拠となったドキュメントを必ず明示させる。
2. **知識の有効期限（TTL）の設定：** 「天（時機）」を逃した古い情報をAIに参照させない自動パージ機能。
3. **ガードレール（検閲）：** 回答が社内規定（法）に反していないか、出力直前に別のAIでチェックする「監視間」の設置。

# SAのアドバイス

孫子は「間（スパイ）ほど親密で、秘匿され、報酬の高いものはない」と言いました。

* AI活用において、データは単なる「材料」ではなく、戦況を左右する「諜報員」です。
* **「どの情報をAIに渡すか」の選別こそが、アーキテクトが担うべき「情報の統治（ガバナンス）」です。**

> **「偽の情報に踊らされる将は敗北し、真実の情報を握る将のみが不敗を得る。」**

>「SAとは、システムアナリスト（Systems Analyst）またはシステムアーキテクト（System Architect）。
