# 結論

**成功しているAIサービスは、単に技術が優れているから勝ったのではありません。「五事」という5つの前提条件が、奇跡的あるいは意図的に噛み合っていたからこそ勝っているのです。**

他社の成功事例を表面的な機能（What）だけで真似しても失敗するのは、その裏側にある「五事（戦略的条件）」をコピーできていないからです。

# 成功を構造で理解するための「五事逆算シート」

成功事例を分析する際は、以下の5つの問いを用いて「勝因の構造」を分解します。

| 評価軸           | 逆算の問い（分析の視点）               | 成功の共通パターン                              |
| ------------- | -------------------------- | -------------------------------------- |
| **道 (Why)**   | なぜこのAIはユーザーに「不可欠」と思われたか？   | 業務の本質的なペインポイントに「AIである必然性」が合致している。      |
| **将 (Who)**   | 誰が現場の不確実性を引き受け、信頼を勝ち取ったか？  | 現場に精通したリーダーが、AIの限界を認めつつ責任を明確にしている。     |
| **天 (When)**  | なぜ「今」この技術スタックで成立したのか？      | モデルのコスト、精度、社会的な受容性のバランスが損益分岐点を超えた。     |
| **地 (Where)** | なぜその組織のデータと文化で実戦投入できたのか？   | すでにデータが整理されているか、AIを受け入れる「素地」が現場にあった。   |
| **法 (How)**   | なぜ一度も致命的な事故を起こさずに運用できているか？ | 高度な全自動化ではなく、人間が介入しやすい「不完全さを許容する設計」がある。 |

# 成功事例を学ぶ際の「孫子的」ポイント

### 1. 精度（Performance）より不敗（Resilience）を見る

成功しているサービスは、意外にもAIの精度が100%ではないことが多いものです。その代わり、AIが間違えた時のリカバー（法）や、人間のフォロー体制（将）が完璧に設計されています。

### 2. 機能（What）より土壌（Ground）を見る

「どんな機能か」よりも、「どんなデータ基盤（地）」の上で動いているかを確認してください。土壌が腐っていれば、どんなに良い種（AIモデル）を撒いても芽は出ません。

### 3. 流行（Trend）より機微（Context）を見る

その成功事例が、なぜ3年前ではなく「今（天）」なのかを考えます。法規制の緩和や、特定のライブラリの登場など、**外部変数の変化を味方につけている**はずです。

# 実務での活用法

競合他社や先行事例を調査し、経営層に報告する際は、機能比較表を作るのをやめましょう。代わりに、この「五事逆算マップ」を提示してください。

> **「競合が成功したのはこの機能のおかげではありません。この『五事』の条件が揃っていたからです。我が社で真似をするなら、まずこの『地（データ）』を整えるところから始めるべきです」**

このように語ることが、アーキテクトとしての「計（計算）」の重みとなります。


# 一文まとめ

> **成功事例とは、真似るべき「答え」ではなく、**
> **五事で分解し、自社の条件と照らし合わせるための「式」である。**
